## 一、项目目标

开发一款基于 **C++20 + Qt6 Widgets** 的现代化 IM 桌面客户端。

项目定位参考：

- 企业微信
- 飞书
- Slack
- Discord

但 **不以复刻这些产品为目标**，而是实现一个具有良好工程架构、代码质量和可扩展性的企业级客户端。

整个项目应始终保持：

- 模块化
- 易维护
- 易扩展
- 可持续迭代

------

# 二、本次开发（V1）目标

本次仅开发 **V1**。

请不要提前实现 V2、V3 的功能。

V1 的目标是完成一个可以正常使用的 IM 客户端基础版本。

------

## 三、V1 功能范围

### 1、用户系统

实现：

- 登录
- 注册
- 自动登录
- Token 本地保存
- 退出登录

------

### 2、联系人系统

实现：

- 好友列表
- 添加好友
- 删除好友
- 搜索好友

------

### 3、聊天系统

实现：

- 单聊
- 文本消息发送
- 消息接收
- 历史消息展示
- 最近会话列表
- 未读消息数量
- 消息时间显示

------

### 4、本地数据

使用 SQLite 保存：

- 用户信息
- 联系人
- 最近会话
- 聊天记录
- 客户端配置

客户端重启后能够恢复数据。

------

### 5、网络通信

使用：

- TCP
- Protocol Buffers

实现：

- 登录请求
- 联系人同步
- 消息发送
- 消息接收

使用自定义二进制协议。

不使用 JSON 作为通信协议。

------

### 6、客户端界面

采用 Qt6 Widgets。

实现：

- 登录窗口
- 主窗口
- 联系人面板
- 会话列表
- 聊天窗口
- 设置页面

UI 保持现代、简洁即可。

无需复杂动画。

------

# 四、本版本明确不做

V1 不实现以下内容：

- 群聊
- 图片消息
- 文件传输
- 音视频通话
- 视频会议
- 屏幕共享
- 消息撤回
- 消息已读
- AI 助手
- 云同步
- 插件系统
- 多端同步
- QML
- Web 前端
- 移动端

这些功能将在后续版本逐步加入。

# 五、技术方案

如果没有特殊说明，请采用成熟、稳定、易维护的方案。

建议技术栈如下：

开发语言：

- C++20

GUI：

- Qt6 Widgets

构建：

- CMake

网络：

- Standalone Asio

通信协议：

- Protocol Buffers

数据库：

- SQLite

日志：

- spdlog

配置：

- yaml-cpp

JSON：

- nlohmann/json（仅用于配置或调试）
